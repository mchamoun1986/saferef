## FIX -- Things that need fixing

### Critical

*(none)*

### High

**1. Homepage outdated product count**
- **Severity**: High
- **Location**: `src/lib/i18n-common.ts` lines 19-21 (EN), 40-43 (FR), 62-64 (SV), 80+ (DE, ES)
- **Issue**: Shows "227 Products" (all 5 languages) — actual count is 135. Shows "36 Refrigerants" — actual count is 47.
- **Fix**: Update to "135 Products" / "135 Produits" / "135 Produkter" etc. and "47 Refrigerants" / "47 Refrigerants" etc. in all language blocks.

**2. Admin products compatible filter dropdown broken**
- **Severity**: High
- **Location**: `src/app/admin/products/page.tsx` lines 362-366
- **Issue**: Filter dropdown options are old short names ("MIDI", "X5", "RM", "AQUIS", "ALL"). The actual `compatibleFamilies` values in the DB are full names ("GLACIAR MIDI", "X5 Transmitter", etc.). Filter matches 0 products for all options.
- **Fix**: Update dropdown options to match real family names: "GLACIAR MIDI", "X5 Direct Sensor Module", "X5 Remote Sensor", "X5 Transmitter", "GLACIAR Controller 10", "GLACIAR RM". Also update placeholder text on lines 514, 601 from `["MIDI","X5"]` to `["GLACIAR MIDI","X5 Transmitter"]`.

### Medium

**3. TypeScript error in products API route**
- **Severity**: Medium
- **Location**: `src/app/api/products/route.ts` line 142
- **Issue**: `Record<string, unknown>` is not assignable to Prisma's `ProductCreateInput`. The spread-based object building loses type safety.
- **Fix**: Type the `data` object explicitly as `Prisma.ProductCreateInput` or cast it.

**4. Stale .next/types references deleted compatibility page**
- **Severity**: Medium
- **Location**: `.next/types/validator.ts` line 95-99
- **Issue**: The .next cache still references `admin/compatibility/page.js` which was deleted. Causes a TS error on `--noEmit`.
- **Fix**: Delete `.next` and rebuild (`rm -rf .next && npm run build`), or just ignore since it is a cache artifact.

### Low

**5. relation-types.ts still imported (legacy code)**
- **Severity**: Low
- **Location**: `src/lib/m2-engine/relation-types.ts` (file itself), imported by: `selection-engine.ts`, `select-controller.ts`, `engine/types.ts`, `relation-selection.test.ts`
- **Issue**: relation-types.ts is legacy code. The V2 engine (SystemDesigner) does not use it, but the old `selectProducts()` function in selection-engine.ts still does.
- **Fix**: Once old selectProducts() is removed, delete relation-types.ts and its test. Not urgent since it causes no runtime issues.

**6. Admin products form placeholder text uses old family names**
- **Severity**: Low
- **Location**: `src/app/admin/products/page.tsx` lines 514, 601
- **Issue**: Placeholder text says `["MIDI","X5"] or ["ALL"]` — should show V2 family names as examples.
- **Fix**: Change to `["GLACIAR MIDI","X5 Transmitter"]` or similar.

**7. 27 accessories have no product image**
- **Severity**: Low
- **Location**: Database (Product table)
- **Issue**: 27 accessories (calibration kits, cable glands, protection brackets, UPS units, X5 small parts) have `image: null`. Not a bug, but would improve the catalogue appearance.
- **Fix**: Add images for the most visible accessories (calibration kits, UPS, backboxes). Lower priority for cable glands and stopping plugs.

**8. LED Sign has price = 0**
- **Severity**: Low
- **Location**: Product code `6100-0002` (LED Sign - Refrigerant Alarm)
- **Issue**: Price is 0 EUR. If included in a BOM, it would flag `hasNaPrice: true`.
- **Fix**: Set the actual price when available, or mark as `status: planned` if not yet quotable.
